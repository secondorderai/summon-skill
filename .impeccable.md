## Design Context

### Users
Summon is for developers and tech-savvy personal AI-agent users evaluating or using repo-native agent workflows.
They arrive with enough technical fluency to judge claims quickly and want to see concrete proof, inspectable output, and a credible path into the GitHub repository.
The main job to be done is to understand what Summon does, trust that it fits real codebases, and click through to the GitHub repo for deeper evaluation or installation.

### Brand Personality
Core personality: Personalized, Self-Evolving, Compound Engineering.
The interface should evoke confidence first, usefulness second, and delight as a subtle supporting quality.
The voice should feel precise, premium, technical, and calm rather than playful, noisy, or over-marketed.

### Aesthetic Direction
Premium minimal, repo-first, and editorial rather than generic SaaS.
Reference: https://impeccable.style/ for refinement, restraint, and strong typographic confidence.
Visual direction should favor warm neutrals, restrained but purposeful accent color, high-quality typography, and clear proof-oriented composition.
Avoid AI-slop aesthetics: no purple-blue gradient tropes, no glow-heavy dark mode, no glassmorphism-as-default, no repetitive card-grid filler, and no fake “futuristic” decoration.

### Design Principles
1. Lead with proof, not explanation. Concrete repo-aware behavior should appear before broad claims.
2. Keep GitHub as the dominant action. Secondary actions should support evaluation, not compete with it.
3. Make everything inspectable. Typography, layout, code examples, and copy should reinforce trust and technical credibility.
4. Use delight sparingly and elegantly. Motion and accent color should reward attention without becoming the point of the page.
5. Preserve premium restraint. Strong hierarchy and bold focal points are good; clutter, redundancy, and trend-driven effects are not.
